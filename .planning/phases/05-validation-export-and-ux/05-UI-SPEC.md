## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge inner padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`, `gap-6`) |
| xl | 32px | Layout gaps (`gap-8`) |
| 2xl | 48px | Major section breaks (not used in this phase) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Source: codebase scan — existing components use `p-2/p-3/p-4/p-6`, `gap-2/gap-4/gap-6/gap-8`. Scale is consistent.

Exceptions:
- Validation error messages: `mt-1` (4px top margin) below offending field — keeps error visually bound to the field without disrupting form flow.
- Touch targets on day-toggle buttons: minimum 36px tall (`py-2` + content) — acceptable for desktop-first academic tool.

Additional semantic colors (already established, continued in Phase 5):
- `amber-*`: Recovery session row tint, holiday warning banner, recovery info footer card

## Interaction States

### Inline Validation (CORT-01)

Trigger: field loses focus (onBlur) OR user attempts to interact with schedule area while form is invalid.

Validated fields and their error conditions:

| Field | Error Condition | Error Message |
|-------|----------------|---------------|
| Total Horas | value <= 0 or empty | "Ingresa un valor mayor a 0" |
| Hrs / Sesión | value <= 0 or empty | "Ingresa un valor mayor a 0" |
| Fecha Inicio | empty | "Selecciona una fecha de inicio" |
| Días de Clase | no day selected | "Selecciona al menos un día de clase" (shown below the days grid) |
| Min. Recuperación | value < 0 or empty | "Ingresa 0 o más minutos" |

Visual treatment per invalid field:
- Input border: `border-rose-400 dark:border-rose-500` (replaces default `border-slate-200 dark:border-slate-700`)
- Error text: `<p className="text-xs text-rose-500 dark:text-rose-400 mt-1">{message}</p>` immediately below the field
- Field label: unchanged (do not add asterisk or change label color)

Schedule suppression: when any validation error is active, `useSchedule` returns an empty array (or the schedule area renders the existing empty state). No new "disabled button" needed — the reactive schedule already disappears when inputs are cleared.

### New Form Fields (EXPO-01, CORT-04)

All new fields follow the existing pattern exactly:

```
label: text-sm font-bold text-slate-500 dark:text-slate-400 uppercase tracking-wider
input: w-full px-4 py-3 rounded-xl border border-slate-200 dark:border-slate-700
       bg-slate-50 dark:bg-slate-800 focus:ring-2 focus:ring-indigo-500 outline-none transition-all
```

New fields to add to `CourseForm`, in order after "Nombre del Curso":

1. **Semestre** — `type="text"`, placeholder `"Ej. 2do Semestre 2026"`, label `"SEMESTRE"`
2. **Nombre Profesor/a** — `type="text"`, placeholder `"Ej. María González"`, label `"NOMBRE PROFESOR/A"`
3. **Email de Contacto** — `type="email"`, placeholder `"Ej. m.gonzalez@universidad.cl"`, label `"EMAIL DE CONTACTO"`
4. **Min. Extra Recuperación** — `type="number"`, default `30`, min `0`, label `"MIN. EXTRA RECUPERACIÓN"` — placed in the existing 2-column grid with "Ses. Recuperación"

### Export Metadata Block (EXPO-02, EXPO-03)

**Excel (EXPO-02):** The metadata block prepends 5 rows before the session data rows:

```
Row 1: ["CRONOGRAMA DE CURSO:", "{courseName}"]
Row 2: ["Semestre:", "{semester}"]
Row 3: ["Profesor/a:", "{professorName}"]
Row 4: ["Email:", "{contactEmail}"]
Row 5: ["Generado:", "{new Date().toLocaleDateString()}"]
Row 6: [] (blank separator)
Row 7: ["Sesión", "Fecha", "Día", "Tipo", "Horas Crono", "Horas Curso", "Acumuladas", "Notas"] (headers)
```

Empty metadata fields: render as empty string (do not write "N/A" or "—").

**Print/PDF (EXPO-03):** A `print-only` div (class `hidden print:block`) is inserted above the schedule table with the following structure:

```
<div class="print-only hidden print:block mb-6 pb-4 border-b-2 border-slate-300">
  <h1 class="text-2xl font-bold">{courseName || "Cronograma de Clases"}</h1>
  <div class="grid grid-cols-2 gap-x-8 gap-y-1 mt-2 text-sm">
    <div><span class="font-bold uppercase text-xs text-slate-500 tracking-wider">Semestre</span><p>{semester}</p></div>
    <div><span class="font-bold uppercase text-xs text-slate-500 tracking-wider">Profesor/a</span><p>{professorName}</p></div>
    <div><span class="font-bold uppercase text-xs text-slate-500 tracking-wider">Email</span><p>{contactEmail}</p></div>
    <div><span class="font-bold uppercase text-xs text-slate-500 tracking-wider">Generado</span><p>{new Date().toLocaleDateString()}</p></div>
  </div>
</div>
```

Empty metadata fields in print: render the field row only if the value is non-empty (skip empty rows).

## Print / PDF Surface Contract

Print CSS already established in `src/styles/index.css`. Phase 5 additions:

- `.print-only` class: `display: none` in screen view; `display: block` in `@media print` — already declared in `index.css` but needs the metadata div to use it.
- Print metadata div placement: inside the `lg:col-span-8` schedule container, before the `<section>` containing stats cards — this ensures it prints at the top of the output page.
- Print page background: `white`, text `black` (already enforced by existing print rules).
- Metadata border separator in print: `border-b-2 border-slate-300` — visible in print because `print-color-adjust: exact` is already set.
